# Release Notes — v0.8.8

## IEEE Citation Formatting

The 33 hand-written IEEE segment builder functions have been replaced with a config-driven template system. Each reference type is now a declarative template array interpreted by a single `buildSegments()` engine. Citation output is unchanged — this is a pure internal refactoring for maintainability.

New exports: `buildSegments`, `TSegmentInstruction`, and all 33 `*_TEMPLATE` constants (e.g., `BOOK_TEMPLATE`, `JOURNAL_ARTICLE_TEMPLATE`) are available for direct use.

## Standalone Evaluation and Validation Functions

The argument evaluation and validation logic that previously lived only as methods on `ArgumentEngine` is now also available as standalone functions:

- `evaluateArgument(ctx, assignment, options?)` — evaluate an argument under a variable assignment
- `checkArgumentValidity(ctx, options?)` — exhaustive truth-table validity check
- `propagateOperatorConstraints(ctx, assignment)` — operator constraint propagation
- `validateArgument(ctx)` — comprehensive invariant validation
- `validateArgumentEvaluability(ctx)` — pre-evaluation structural validation
- `validateArgumentAfterPremiseMutation(ctx)` — lightweight post-mutation check
- `collectArgumentReferencedVariables(ctx)` — variable index by ID and symbol

Each function accepts a narrow read-only context interface (`TArgumentEvaluationContext` or `TArgumentValidationContext`), making them usable without constructing a full `ArgumentEngine`.

The existing `ArgumentEngine` methods are unchanged — they now delegate to these standalone functions internally.
