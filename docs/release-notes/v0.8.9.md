# Release Notes

## Auto-Normalize: Smarter Formula Collapse

Formula nodes now collapse automatically when they serve no grouping purpose. A formula is only justified if its subtree contains a binary operator (`and`/`or`). Formulas wrapping only variables or `not` chains are collapsed by promoting their child.

### What changed

- **Formula collapse rule:** Formulas without a binary operator in their bounded subtree are automatically collapsed during expression removal, snapshot/data loading, and explicit `normalize()` calls.
- **Operator collapse gated on `autoNormalize`:** Both operator collapse (0/1 children) and formula collapse only run when `grammarConfig.autoNormalize` is `true`. When `false`, the tree is free-form and can be in any state.
- **Default `autoNormalize` changed to `true`:** The default grammar config now enables auto-normalization. This preserves existing collapse behavior and adds formula collapse.
- **New `normalize()` API:** Call `normalizeExpressions()` on a premise or `normalizeAllExpressions()` on an argument engine to normalize on demand, regardless of the `autoNormalize` setting. Useful for callers who build trees freely and then normalize.
- **Post-load normalization:** `fromSnapshot` and `fromData` now run a normalization sweep after loading when `autoNormalize` is `true`, catching unjustified formulas in serialized data.
