# 0.5.9

## Bug Fixes

- **checksumConfig Set reconstruction:** `fromSnapshot`, `fromData`, and `rollback` now automatically convert array-serialized `checksumConfig` fields back to `Set<string>` instances during deserialization. Previously, JSON round-tripping a snapshot would leave these fields as plain arrays, violating the `TCoreChecksumConfig` type contract.

## New Exports

- **`normalizeChecksumConfig(config)`** — Standalone utility that ensures all `TCoreChecksumConfig` fields are `Set<string>`, converting from arrays or other iterables as needed. Called internally by `fromSnapshot`/`fromData`/`rollback`, but also exported for consumers who deserialize checksum configs independently.
