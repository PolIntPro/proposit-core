# 0.5.11

## Bug Fixes

- **Nested checksumConfig normalization:** `normalizeChecksumConfig` was only applied at the `ArgumentEngine` level during `fromSnapshot`/`rollback`, but nested configs inside premise and expression snapshots were passed through un-normalized. After a native `JSON.stringify` round-trip, `ExpressionManager.attachChecksum` would throw `TypeError: fields is not iterable`. `PremiseEngine.fromSnapshot` and `ExpressionManager.fromSnapshot` now normalize their own configs independently.
